# Second Brain Globe

An interactive 3D knowledge globe. Every cluster of thought, tethered to a single core.

![Second Brain Globe hero](docs/hero.png)

![Demo](docs/demo.gif)

A dark wireframe globe holds twelve glowing clusters of knowledge. At the center, a nested-cube tesseract nucleus convolves and pulses, feeding energy along curved tethers to every cluster. Built with `3d-force-graph`, `three.js` and Vite.

## Features

- **Tesseract nucleus.** A solid red cube inside counter-rotating wire cubes and frosted shells, with a pulsing additive glow. The power source of the network.
- **Force-directed dandelion clusters.** Twelve hubs pinned on a fibonacci sphere, each puffing out 16 to 30 leaves through per-link distances and tuned charge forces.
- **Energy pulses.** Bright particles continuously travel from the core outward along every tether, each with its own seeded speed and offset.
- **Click-to-focus cinematics.** Click a hub and the camera flies in while everything outside the cluster falls away to near-black. Escape restores the globe.
- **Bloom post-processing.** UnrealBloomPass makes the core, hub balls, particles and dust glow while the wireframe stays subtle.
- **Seeded reproducible data.** One seed, one layout. The whole graph and environment are generated from a tiny mulberry32 PRNG.

## Quickstart

```bash
git clone https://github.com/Jose-Gael-Cruz-Lopez/second-brain-globe.git
cd second-brain-globe
npm install
npm run dev
```

## Controls

| Input | Action |
| --- | --- |
| Drag | Rotate the globe |
| Scroll | Zoom in and out |
| Hover a node | Preview its label |
| Click a hub (or leaf) | Fly in and focus that cluster |
| Click empty space / Esc | Release focus and fly home |

## Data

The graph is generated by `src/data/generateBrain.js`, which returns `{ nodes, links }`:

```js
// node
{
  id: 'hub-3',           // unique string
  type: 'hub',           // 'core' | 'hub' | 'leaf' | 'branch'
  cluster: 'Research',   // cluster name (null for the core)
  label: 'Research',     // tooltip text; leaves use 'Research · node 07'
  val: 10,               // visual size for leaves/branches (1, 2 or 5)
  color: '#c8b6ff',      // node color from the palette
  weight: 1.12,          // hubs only, scales the hub ball (0.9 to 1.3)
  fx: 120, fy: -80, fz: 210 // fixed position (core and hubs only)
}

// link
{
  source: 'hub-3',       // node id
  target: 'hub-3-leaf-6',
  kind: 'spoke',         // 'tether' | 'spoke' | 'branch'
  cluster: 'Research',   // cluster the link belongs to
  distance: 42.5,        // force distance (precomputed, seeded)
  curveRotation: 3.1,    // tethers only, radians
  particleSpeed: 0.006,  // tethers only
  particleOffset: 0.4    // tethers only, 0..1 starting offset
}
```

To swap in real data, replace the `generateBrain(42)` call in `src/main.js` with any JSON matching this schema. Exports from Obsidian, Notion, or a Supabase table all work: map your notebooks or databases to hubs, and your notes or rows to leaves. Keep one `core` node and pin your hubs (set `fx/fy/fz` on a sphere shell) for the signature silhouette.

## Deploy

**Vercel** works zero-config for Vite: import the repo and deploy, no settings needed.

**GitHub Pages** needs the base path set in `vite.config.js`:

```js
export default defineConfig({
  base: '/second-brain-globe/',
  resolve: { dedupe: ['three'] },
});
```
